## Auto-Resizing

By default, Dockview uses a `ResizeObserver` to automatically detect when the container element changes size and updates the layout accordingly. This requires no configuration.

If you need to manage sizing yourself, for example when embedding Dockview inside a container that doesn't produce `ResizeObserver` notifications, you can disable auto-resizing and call `api.layout()` manually.

```ts
// Disable the internal ResizeObserver
disableAutoResizing={true}

// Then manually drive layout updates:
api.layout(width, height);
```

<FrameworkSpecific framework='JavaScript'>
```ts
const api = createDockview(element, {
    theme: themeAbyss,
    disableAutoResizing: true,
    createComponent: (options) => new MyPanel(),
});

// Call whenever the container size changes
api.layout(element.clientWidth, element.clientHeight);
```
</FrameworkSpecific>

<FrameworkSpecific framework='React'>
```tsx
const apiRef = React.useRef<DockviewApi>();

const onReady = (event: DockviewReadyEvent) => {
    apiRef.current = event.api;
};

// Trigger a resize when needed:
apiRef.current?.layout(width, height);

return (
    <DockviewReact
        disableAutoResizing={true}
        onReady={onReady}
        components={components}
    />
);
```
</FrameworkSpecific>

<FrameworkSpecific framework='Vue'>
```ts
methods: {
    onReady(event: DockviewReadyEvent) {
        this.api = event.api;
    },
    resize(width: number, height: number) {
        this.api?.layout(width, height);
    },
},
template: `<dockview-vue :disableAutoResizing="true" @ready="onReady" />`,
```
</FrameworkSpecific>

<FrameworkSpecific framework='Angular'>
```ts
export class AppComponent {
    private api: DockviewApi;

    onReady(event: DockviewReadyEvent) {
        this.api = event.api;
    }

    resize(width: number, height: number) {
        this.api?.layout(width, height);
    }
}

// In template:
// <dv-dockview [disableAutoResizing]="true" (ready)="onReady($event)" />
```
</FrameworkSpecific>

:::info
You can read the current dimensions at any time via `api.width` and `api.height`.
:::

## Hiding Borders

By default, Dockview renders thin borders between groups in the grid. Set `hideBorders` to remove them:

<FrameworkSpecific framework='JavaScript'>
```ts
const api = createDockview(element, {
    theme: themeAbyss,
    hideBorders: true,
    createComponent: (options) => new MyPanel(),
});
```
</FrameworkSpecific>

<FrameworkSpecific framework='React'>
```tsx
<DockviewReact hideBorders={true} onReady={onReady} components={components} />
```
</FrameworkSpecific>

<FrameworkSpecific framework='Vue'>
```ts
template: `<dockview-vue :hideBorders="true" @ready="onReady" />`
```
</FrameworkSpecific>

<FrameworkSpecific framework='Angular'>
```ts
// In template:
<dv-dockview [hideBorders]="true" (ready)="onReady($event)" />
```
</FrameworkSpecific>

## Updating Options at Runtime

`api.updateOptions()` lets you change any option after the component has initialized. Only the provided keys are updated; others remain unchanged.

```ts
api.updateOptions({
    hideBorders: true,
    locked: false,
    theme: themeLight,
});
```
